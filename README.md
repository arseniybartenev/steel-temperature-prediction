## Описание проекта

Данный проект выполнен в рамках дипломной работы по курсу Data Science. Задача — разработать модель машинного обучения для прогнозирования температуры стали в процессе выплавки на металлургическом комбинате «Стальная птица». Точный прогноз температуры позволяет оптимизировать энергозатраты, сократить время плавки и повысить качество продукции.

**Цель:** Построить модель регрессии, предсказывающую итоговую температуру стали на основе технологических параметров процесса. Требуемая точность — **MAE < 6.8°C**.

## Данные

В распоряжении имеются синтезированные данные, имитирующие реальный производственный процесс. Данные состоят из нескольких файлов, содержащих информацию о различных этапах обработки стали в ковше:

- `data_arc_new.csv` – данные об электродах (время нагрева, активная и реактивная мощность);
- `data_bulk_new.csv` – объёмы подаваемых сыпучих материалов (15 типов);
- `data_bulk_time_new.csv` – время подачи сыпучих материалов *(не используется)*;
- `data_gas_new.csv` – объём продувки инертным газом;
- `data_temp_new.csv` – результаты измерения температуры (целевая переменная);
- `data_wire_new.csv` – объёмы подаваемых проволочных материалов (9 типов);
- `data_wire_time_new.csv` – время подачи проволоки *(не используется)*.

Ключевым полем для объединения данных является `key` – номер партии. В файлах может быть несколько строк с одним `key`, соответствующих разным итерациям обработки.

Целевой признак — **температура стали перед выпуском**.

## Этапы работы

1. **Исследовательский анализ данных (EDA)**
   - Анализ пропусков, выбросов, дубликатов.
   - Визуализация распределений признаков и их взаимосвязей.
   - Корреляционный анализ с использованием коэффициента PhiK для смешанных типов данных.

2. **Предобработка данных**
   - Приведение названий столбцов к единому стилю snake_case.
   - Преобразование временных меток в формат `datetime`.
   - Заполнение пропусков в данных о добавках нулями (отсутствие добавки).
   - Удаление неинформативных признаков (файлы времени подачи).

3. **Feature Engineering**
   - Агрегация данных по ключу `key`: суммирование энергетических показателей, общего объёма сыпучих материалов, проволоки и газа.
   - Генерация новых признаков: полная мощность, длительность нагрева, активная энергия, суммарный объём сыпучих материалов.
   - Отбор признаков на основе корреляционного анализа.

4. **Моделирование**
   - Разделение данных на обучающую и тестовую выборки (75/25).
   - Установление baseline с помощью `DummyRegressor`, линейных моделей и дерева решений.
   - Обучение моделей: Random Forest, LightGBM, CatBoost с подбором гиперпараметров.
   - Оценка качества по метрике MAE.

5. **Интерпретация модели**
   - Анализ важности признаков.
   - Использование SHAP для объяснения предсказаний лучшей модели.

## Используемые технологии

- **Python** — основной язык.
- **Pandas**, **NumPy** — обработка данных.
- **Matplotlib**, **Seaborn** — визуализация.
- **Phik** — корреляционный анализ.
- **Scikit-learn** — предобработка, baseline-модели, метрики, валидация.
- **LightGBM**, **CatBoost** — градиентный бустинг.
- **SHAP** — интерпретация моделей.
- **Jupyter Notebook** — среда разработки.

## Результаты

| Модель              | MAE (тест) |
|---------------------|------------|
| DummyRegressor      | ~15.0      |
| Linear Regression   | ~12.5      |
| Decision Tree       | ~9.8       |
| Random Forest       | ~7.2       |
| LightGBM            | ~6.5       |
| **CatBoost**        | **6.1**    |

Лучшей моделью оказался **CatBoostRegressor** с оптимизированными гиперпараметрами.  
Достигнутое значение MAE на тестовой выборке составило **6.1°C**, что удовлетворяет требованию заказчика (MAE < 6.8).  

**Ключевые факторы**, влияющие на температуру (по результатам SHAP):
- Суммарная активная энергия,
- Время нагрева,
- Объём подаваемого сыпучего материала bulk_12,
- Количество продувки газом.

Разработанная модель может быть внедрена в систему управления технологическим процессом для:

- Корректировки режимов плавки в реальном времени;
